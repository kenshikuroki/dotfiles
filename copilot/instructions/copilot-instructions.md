# 個人用グローバル指示

## 対話スタイル
- 説明は日本語で簡潔に。結論 → 根拠の順。
- 不確実な内容には「推測」「未検証」と明示。
- 冗長な前置き・謝罪・確認の繰り返しは避ける。

## エンジニアリング方針
- コードのコメント・識別子は英語。
- 可読性を優先しつつ、計算コストが支配的な箇所(内側ループ、大規模ループ等)はその旨をコメントで明記する。
- 外部依存の追加(ライブラリ導入、パッケージインストール等)は提案のみ行い、実行前に確認する。
- エラーは握りつぶさず、文脈情報(変数値、発生箇所など)を含めて報告・ログする。
- マジックナンバーは避け、定数化または由来をコメントで示す。
- コード・ワークフロー上の改善点は、指摘されなくても積極的に提案する。

## 環境前提
- WSL2 + Ubuntu 24.04.4
- Shell: zsh
- Editor: VS Code
- 時々VS codeのRemote-SSHでLinuxサーバに繋いで作業。
- 物理計算コードでは自然単位系($\hbar = c = 1$)を前提とする。
